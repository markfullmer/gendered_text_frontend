Instructions

1. The Word Map

Review the full list of currently available gendered words with mappings in the code. If you have suggestions for more words to add, contact us.

2. Add Placeholders

Text must have personae gender indicators entered as placeholder text, as well as a legend that instructs the code which gender should be used for each persona. Use our on-line tool to swap gendered text for placeholder text.

For example, original the Alice in Wonderland sentence, would be modified and a legend inserted:

When the Rabbit came near {{ her(Alice) }}, {{ Alice }} began, in a low, timid voice, "If you please, {{ sir(Rabbit) }}—" The Rabbit started violently, dropped the white kid-gloves and the fan and scurried away into the darkness as hard as {{ he(Rabbit) }} could go.

[[Alice/Alan/Al:female]]

** Note the use of "herd" to differentiate determiner & object. See Callouts, below.

2a. Texts with multiple personae

The sentence from D.H Lawrence's Sons and Lovers, "He kept her because he never satisfied her", illustrates two personae working in tandem. The text would be parameterized as:

{{ He(Paul) }} kept {{ her(Clara) }} because {{ he(Paul) }} never satisfied {{ her(Clara) }}.

Adding the legend [[Paulina/Paul/Paulie:male][Clara/Clark/Clax:female]] would generate the original text as default, while [[Paulina/Paul/Paulie:female][Clara/Clark/Clax:male]] would reverse the roles:

She kept him because she never satisfied him.

and [[Paulina/Paul/Paulie:male][Clara/Clark/Clax:male]] would generate:

He kept him because he never satisfied him.

3. Create the Legend

Text to be processed must include a legend, wrapped in double square brackets, which may appear anywhere in the text. An example legend:

[[Mindy/Mork/Mod:male][Charlize/Charles/Char:female][Kate/Ken/Kan:non-binary]]

Each element in the legend is wrapped in square brackets, and consists of the gendered proper names for the character, in the order FEMALE/MALE/NON-BINARY.

The default gender of that personae is specified after a colon.

So, using the legend above, the program will look for a character named "Mindy" or "Mork" or "Mod" and render this character as "Mork" with the male pronoun set. It will look for a character named "Charlize" or "Charles" or "Char" and render the character as "Charlize" with the female pronoun set. It will look for a character named "Kate," "Ken" or "Kan," and render the character as "Kan" with a non-binary pronoun.

Callouts

  • An idiosyncrasy of English language pronouns is that, while the male possessive pronoun "his" (as in "I like his shirt") is identical to the determinative (as in "That shirt is his"), the female equivalents differ (i.e., "I like her shirt" vs. "That shirt is hers"). Accordingly, male determinative pronouns must be entered as "hisd" (e.g., That shirt is {{ hisd(Mork) }}) for differentiation from the possessive "his" (to avoid the result "I like hers shirt").
  • Similarly, the female determiner ("I like her hairdo") is identical to the female object ("I like her"), but the male forms differ ("I like his hairdo" vs. "I like him"). Therefore, the female determiner must be identified as "herd".
